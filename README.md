# 30-Day JavaScript Interview Prep Checklist

## Week 1: Core Foundations
- [ ] **Day 1:** Variables, data types, operators  
- [ ] **Day 2:** Functions (declaration, expression, arrow functions)  
- [ ] **Day 3:** Scope, closures, hoisting  
- [ ] **Day 4:** Arrays & objects (methods, destructuring)  
- [ ] **Day 5:** Practice 5 easy problems (strings, arrays)  
- [ ] **Day 6:** Review and revise concepts  
- [ ] **Day 7:** Practice 10–15 LeetCode easy JS problems  

---

## Week 2: Advanced Language Features
- [ ] **Day 8:** Prototypes, inheritance, `this` keyword  
- [ ] **Day 9:** ES6+ features (spread/rest, template literals, default params)  
- [ ] **Day 10:** Promises, async/await, event loop  
- [ ] **Day 11:** Error handling, try/catch, custom errors  
- [ ] **Day 12:** Modules (import/export), CommonJS vs ES Modules  
- [ ] **Day 13:** Practice medium-level problems (string/array manipulation)  
- [ ] **Day 14:** Weekly mock interview session  

---

## Week 3: Browser & System Concepts
- [ ] **Day 15:** DOM manipulation, event bubbling/capturing  
- [ ] **Day 16:** Fetch API, AJAX, JSON handling  
- [ ] **Day 17:** LocalStorage, SessionStorage, cookies  
- [ ] **Day 18:** Debouncing, throttling, performance optimization  
- [ ] **Day 19:** Security basics (XSS, CSRF, CORS)  
- [ ] **Day 20:** Mock interview-style coding session  

---

## Week 4: Problem-Solving & Mock Interviews
- [ ] **Day 21:** Algorithms (sorting, searching, recursion)  
- [ ] **Day 22:** Data structures (stack, queue, linked list in JS)  
- [ ] **Day 23:** System design basics (API design, scalability concepts)  
- [ ] **Day 24:** Practice hard-level problems (graphs, dynamic programming)  
- [ ] **Day 25:** Behavioral interview prep (STAR method)  
- [ ] **Day 26:** Mock interview with timed coding challenges  
- [ ] **Day 27:** Review weak areas, revisit tricky problems  
- [ ] **Day 28:** Full mock interview simulation  
- [ ] **Day 29:** Final revision of all topics  
- [ ] **Day 30:** Confidence check and relaxation before interviews  

---

## Notes
- Aim for **2–3 coding problems daily**  
- Keep a **flashcard deck** for closures, async/await, prototypes  
- End each week with a **mock interview or timed practice**  
- Track progress by **checking off each day’s task**
