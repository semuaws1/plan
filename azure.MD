# 30-Day Azure Learning Plan (Solution Architect Track)

A structured 30‑day Azure learning plan tailored for aspiring Solution Architects.  
It balances fundamentals, hands‑on labs, and architecture thinking to build both technical depth and design perspective.

---

## 📅 Week 1: Foundations
**Goal:** Understand Azure basics, core services, and governance.

- **Day 1–2**
  - Explore Azure Portal, Resource Groups, Subscriptions
  - Learn about Azure Regions and Availability Zones
  - *Hands‑on:* Deploy a simple VM and storage account
- **Day 3–4**
  - Study Identity & Access Management (Azure AD, RBAC)
  - *Hands‑on:* Create users, assign roles, configure conditional access
- **Day 5–7**
  - Networking fundamentals: VNets, Subnets, NSGs, VPN Gateway
  - *Hands‑on:* Build a hub‑and‑spoke network topology
  - Read about governance: Azure Policy, Blueprints, Cost Management

---

## 📅 Week 2: Core Compute & Storage
**Goal:** Learn how workloads run and scale in Azure.

- **Day 8–9**
  - Azure Virtual Machines, Scale Sets, Availability Sets
  - *Hands‑on:* Deploy a VMSS with autoscaling
- **Day 10–11**
  - Azure App Services, Functions, Containers (AKS basics)
  - *Hands‑on:* Deploy a sample web app to App Service
- **Day 12–13**
  - Storage options: Blob, Table, Queue, Disk
  - *Hands‑on:* Configure lifecycle management and replication
- **Day 14**
  - Databases: Azure SQL, Cosmos DB, PostgreSQL
  - *Hands‑on:* Deploy Azure SQL with geo‑replication

---

## 📅 Week 3: Security, Integration & Monitoring
**Goal:** Architect secure, observable, and integrated solutions.

- **Day 15–16**
  - Security: Key Vault, Managed Identities, Defender for Cloud
  - *Hands‑on:* Store secrets in Key Vault and integrate with App Service
- **Day 17–18**
  - Integration: Service Bus, Event Grid, Logic Apps
  - *Hands‑on:* Build an event‑driven workflow with Event Grid + Function
- **Day 19–20**
  - Monitoring: Azure Monitor, Log Analytics, Application Insights
  - *Hands‑on:* Configure dashboards and alerts for a sample app

---

## 📅 Week 4: Architecture & Best Practices
**Goal:** Think like a Solution Architect.

- **Day 21–22**
  - Review Well‑Architected Framework (cost, reliability, performance, security, operations)
  - Map workloads against these pillars
- **Day 23–24**
  - High Availability & Disaster Recovery strategies
  - *Hands‑on:* Deploy geo‑redundant storage and test failover
- **Day 25–26**
  - Hybrid & Multi‑Cloud: Azure Arc, ExpressRoute
  - *Hands‑on:* Connect on‑prem resources with Azure Arc
- **Day 27–28**
  - Design patterns: Microservices, Serverless, Event‑driven
  - *Case study:* Architect a scalable e‑commerce solution
- **Day 29–30**
  - Practice exam prep for **AZ‑305: Designing Microsoft Azure Infrastructure Solutions**
  - Review architecture scenarios and whiteboard solutions

---

## 🎯 Key Tips
- Daily rhythm: **1–2 hours study + 1 hour hands‑on lab**  
- Documentation first: Always read **Microsoft Learn** modules before labs  
- Architecture mindset: Don’t just deploy—ask *why this service, what trade‑offs, how does it scale?*  
- End goal: By Day 30, you should be comfortable designing end‑to‑end solutions and ready to attempt **AZ‑305 certification**
